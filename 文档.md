## 项目定位

- 以 Mintlify Writer 为参考，重建一个“代码 → 文档”链路的前后端分离应用。
- 后端仍使用 Spring Boot 3.2 + MySQL + Redis，仅保留文档生成、任务调度、反馈统计等核心模块。
- 前端改为 Vue 3 Web 应用（替代 VS Code / JetBrains 插件），支持提交代码、轮询状态并反馈评分。
- 登录、会员、团队、Webhook 等复杂能力暂不实现，数据库只有 `docs`、`api_keys`。

## 核心能力

1. `/docs/write/v3`、`/docs/write/v3/no-selection`：异步任务 + 轮询 `/docs/worker/{id}`，任务落盘后可在服务重启后恢复，协议与原 Node 版兼容。
2. `/docs/feedback`、`/docs/intro`、`/docs/intro/discover`：记录反馈与调研数据。
3. `/api/v1/document`、`/api/v1/list/languages`、`/api/v1/list/formats`：API-Key 鉴权 + Redis 限流的公共接口。
4. Vue 前端：整合登录/注册、代码输入、任务监控、历史检索、管理员面板（用户角色/配额 & API-Key），默认 2 秒轮询，覆盖验收场景。

## 非功能性需求

1. **可靠性**：后台线程池 + 落盘任务 + 模型兜底确保有响应；`/health`、结构化日志便于观测。
2. **性能**：异步任务 + Redis 限流，普通用户还可通过 `apiQuota` 控制提交频率；前端轮询频率可配置。
3. **安全性**：API-Key 鉴权、JWT 登录、角色/配额控制与集中式 CORS 白名单，防止滥用。

## 调整概览

- 删除 `vscode/`、`intellij/` 等插件目录，改为 `frontend/`（Vue 3）。
- `docs/*`、`README` 均更新为“前后端分离 + Web 控制台”描述。
- 新增 `web.cors.*` 配置与 `WebCorsConfig`，方便本地和线上前端调用。
- 前端默认字段覆盖代码/上下文/语言/格式/宽度/反馈等主流程。

## 后续建议

1. 已支持 JWT + 角色控制，后续可进一步扩展组织/团队维度。
2. 为文档生成增加更多模型或本地 LLM 适配，提升离线可用性。
3. 扩充 JUnit 与前端单元测试，覆盖 AI 网关、限流、轮询等关键路径。
